# Trader MD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create two parallel trading paths — `trader/` (Python, renamed from `challenge/`) and `trader_md/` (MD-driven, everything new) — sharing the same income_desk engine.

**Architecture:** `trader/` is a straight rename of `challenge/` with updated imports. `trader_md/` is a new package with parser (reads 5 MD file types), models (dataclasses), and runner (executes parsed workflow plans by calling existing workflow APIs). All MD content files live under `trader_md/`.

**Tech Stack:** Python 3.12, PyYAML, existing income_desk workflow APIs, Pydantic

**Spec:** `docs/superpowers/specs/2026-03-29-trader-md-design.md`

# income_desk/trader_md/models.py
from __future__ import annotations
from dataclasses import dataclass, field


@dataclass
class Gate:
    expression: str          # 'pulse != "RED"'
    on_fail: str             # "HALT", "SKIP", "BLOCK", "ALERT", "WARN"
    message: str = ""        # "Market pulse {pulse} — trading halted"


@dataclass
class Step:
    name: str                # "Market Pulse"
    workflow: str            # "check_portfolio_health"
    inputs: dict[str, str] = field(default_factory=dict)   # field → binding expr
    outputs: dict[str, str] = field(default_factory=dict)  # name → binding expr
    gates: list[Gate] = field(default_factory=list)
    requires: str | None = None        # "live_broker"
    on_simulated: str | None = None    # "WARN ..."


@dataclass
class Phase:
    name: str                # "Market Assessment"
    number: int
    steps: list[Step] = field(default_factory=list)
    requires_positions: bool = False


@dataclass
class BrokerProfile:
    name: str                # "tastytrade_live"
    broker_type: str         # "tastytrade", "dhan", "simulated"
    mode: str = "live"       # "live", "paper"
    market: str = "US"
    currency: str = "USD"
    credentials_source: str = ".env.trading"
    fallback: str = "simulated"


@dataclass
class UniverseSpec:
    name: str
    market: str = "US"
    description: str = ""
    tickers: list[str] = field(default_factory=list)


@dataclass
class RiskProfile:
    name: str
    max_risk_per_trade_pct: float = 3.0
    max_portfolio_risk_pct: float = 30.0
    max_positions: int = 8
    min_pop: float = 0.50
    min_dte: int = 7
    max_dte: int = 45
    min_iv_rank: float = 20.0
    max_spread_pct: float = 0.05
    profit_target_pct: float = 0.50
    stop_loss_pct: float = 2.0
    exit_dte: int = 5
    regime_rules: dict[str, bool] = field(default_factory=lambda: {
        "r1": True, "r2": True, "r3": False, "r4": False,
    })


@dataclass
class WorkflowPlan:
    name: str
    description: str = ""
    broker_ref: str = "simulated"     # references .broker.md
    universe_ref: str = ""            # references .universe.md
    risk_ref: str = "moderate"        # references .risk.md
    phases: list[Phase] = field(default_factory=list)
    # Resolved at runtime
    broker: BrokerProfile | None = None
    universe: UniverseSpec | None = None
    risk: RiskProfile | None = None
```

Parsing rules for .workflow.md:
- YAML frontmatter → WorkflowPlan metadata (name, description, broker_ref, universe_ref, risk_ref)
- `## Phase N: <name>` headers → Phase objects
- `### Step: <name>` headers → Step objects
- Under each step: `workflow:`, `inputs:`, `outputs:`, `gate:`, `on_fail:`, `requires:`, `on_simulated:`
- `requires_positions: true` under a phase header → Phase.requires_positions = True

Parsing rules for .broker.md / .universe.md / .risk.md:
- YAML frontmatter only — all data is in frontmatter
- For .universe.md: tickers can be in frontmatter `tickers:` list OR parsed from `- TICKER` bullet lists in body

`resolve_references()`:
- Looks up `broker_ref` → loads `broker_profiles/<ref>.broker.md`
- Looks up `universe_ref` → loads `universes/<ref>.universe.md`
- Looks up `risk_ref` → loads `risk_profiles/<ref>.risk.md`
- Sets `plan.broker`, `plan.universe`, `plan.risk`

# income_desk/trader_md/runner.py
class TradingRunner:
    def __init__(self, workflow_path: str):
        """Load workflow MD, resolve all references, set up MarketAnalyzer."""

    def run(self, interactive: bool = False) -> ExecutionReport:
        """Execute all phases. For each step:
        1. Resolve input bindings
        2. Build Pydantic request
        3. Call workflow function
        4. Store outputs in context
        5. Evaluate gates
        6. Handle on_fail actions
        7. Print results (tabular)
        """

    def validate(self) -> list[str]:
        """Parse-only check: are all references valid? Gate syntax ok?"""

    def dry_run(self) -> str:
        """Show what would execute without calling APIs."""
```

Key implementation details:
- `_resolve_binding(expr, context)` — resolves `$universe`, `$capital`, `$phase1.iv_rank_map`, `$result.sentinel_signal`, `$risk.min_pop`
- `_evaluate_gate(gate, step_result, context)` — evaluates boolean expressions
- `_build_request(step, context)` — maps resolved inputs to Pydantic request fields
- `_call_workflow(workflow_name, request, ma)` — dynamic dispatch to workflow functions
- Context dict accumulates outputs: `{"phase1": {"pulse": "GREEN", "iv_rank_map": {...}}, ...}`
- Uses `print_signature()` and `print_table()` from `income_desk.trader.support` for output formatting

Broker setup:
- Read `plan.broker.credentials_source` (e.g. `.env.trading`)
- Load dotenv from that file
- Connect broker based on `plan.broker.broker_type` + `plan.broker.mode`
- On failure: check `plan.broker.fallback` — if "simulated", use preset
